Разработай Windows-приложение (Windows 10) для весового учёта и печати этикеток на куриную продукцию.

ТРЕБОВАНИЯ К ПРИЛОЖЕНИЮ:
- Готовый .exe файл, запуск по двойному клику, без установки.
- Локальный сервер (Python/Go/Node) + UI открывается в браузере по умолчанию (localhost) для экономии токенов на отрисовку интерфейса — минимизировать тяжёлые фреймворки.
- Подключение к электронным весам через COM-порт (RS-232/USB-COM), чтение текущего веса в реальном времени, отображение на экране (брутто/нетто).
- Подключение к принтеру этикеток (термопринтер, например через ZPL/TSPL команды или драйвер Windows-печати).

ИНТЕРФЕЙС:
- Окно/страница в браузере с крупным отображением текущего веса (онлайн, обновление 2–5 раз/сек).
- Выбор продукта из списка (Филе куриное, Голень с кожей, Крылышки, Тушка цыпленка и т.д.).
- Поля: дата производства (по умолчанию — текущая дата), срок годности (рассчитывается автоматически по продукту или вводится вручную).
- Кнопка "Печать этикетки".

МАКЕТ ЭТИКЕТКИ (например, 58x40 мм или 60x40 мм, уточнить размер):
- Верх: название продукта крупным шрифтом по центру.
- Левый блок: таблица с весом — нетто и брутто.
- Правый блок: информация о производителе (название, адрес, ИНН/контакты).
- Центр-право: штрихкод (EAN-13 или Code128) для сканирования на кассе.
- Низ: дата производства/упаковки и срок годности.

ТЕХНИЧЕСКИЕ ДЕТАЛИ:
- Язык реализации: [укажи — Python/C#/др.]
- Формат печати: ZPL/TSPL для термопринтеров или системный драйвер.
- Конфигурация COM-порта весов и принтера — через конфиг-файл или настройки в UI.
- Сборка в один .exe (PyInstaller/аналог).

Напиши код приложения с учётом этих требований, начни с архитектуры и выбора стека.
